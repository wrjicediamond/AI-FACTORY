# 电子信息制造业企业智能化项目核心功能模块详细设计 v1

> 定位：核心功能模块详细设计  
> 范围：只描述当前第一版需要落地的核心模块，不展开完整数据库表结构和代码级实现  
> 技术路线：React + ASP.NET Core Web API + SQL Server + Redis + MinIO/OSS + Hangfire  
> 设计目标：说明每个模块做什么、产出什么、与其他模块如何关联，以及后续如何扩展。

## 1. 设计背景

本系统面向电子信息制造业企业智能化建设，核心目标是把企业的模糊业务需求转化为可配置、可测试、可发布、可审计的智能体能力。

第一版不追求复杂多智能体自治，也不追求完整知识库和算法平台，而是优先完成以下闭环：

```text
用户提需求
  -> 开发端整理需求
  -> 配置数据、权限、规则、工具、智能体
  -> 沙箱测试
  -> 发布审核
  -> 用户使用
  -> 结果审计
  -> 反馈沉淀
```

因此，本详细设计按照模块方式展开，每个模块均从“功能职责、主要能力、输入输出、关联模块、扩展方向”五个角度说明。

## 3. 用户与组织模块

### 3.1 模块功能

用户与组织模块负责管理系统中的企业、部门、用户和外部协同平台账号，是权限计算和数据隔离的基础。

核心功能包括：

| 功能 | 说明 |
| --- | --- |
| 租户管理 | 管理接入平台的企业或组织 |
| 组织管理 | 管理企业、部门、产线、班组等组织层级 |
| 用户管理 | 创建、启用、停用、锁定用户 |
| 外部账号绑定 | 绑定飞书 open_id、union_id、钉钉 user_id |
| 用户归属 | 绑定用户所属租户、部门、岗位和角色 |

### 3.2 输入输出

| 类型 | 内容 |
| --- | --- |
| 输入 | 企业信息、部门信息、用户信息、外部账号 ID |
| 输出 | 平台用户、组织结构、用户身份上下文 |

### 3.3 与其他模块的关联

| 关联模块 | 关联方式 |
| --- | --- |
| 角色与权限模块 | 用户绑定角色后参与权限计算 |
| 需求管理模块 | 需求必须关联提交用户和所属租户 |
| 数据权限模块 | 用户所属组织影响可见数据范围 |
| 智能体运行模块 | 每次运行都要识别用户身份和租户 |
| 协同平台模块 | 飞书、钉钉消息必须映射到平台用户 |
| 审计模块 | 审计日志记录操作用户、租户和组织 |

### 3.4 扩展方向

后续可扩展 SSO、LDAP/AD、企业微信、多组织兼职、岗位权限、组织级数据范围、用户生命周期同步。

## 4. 角色与权限模块

### 4.1 模块功能

角色与权限模块是系统安全控制核心，负责控制用户能访问哪些页面、接口、数据、工具和智能体。

核心功能包括：

| 功能 | 说明 |
| --- | --- |
| 角色管理 | 配置系统角色和企业自定义角色 |
| 菜单权限 | 控制用户端、开发端、管理端菜单可见性 |
| API 权限 | 控制后端接口调用权限 |
| 数据权限 | 控制数据源、表、字段访问权限 |
| 工具权限 | 控制 SQL、HTTP、Rule 工具执行权限 |
| 智能体权限 | 控制用户可使用哪些智能体 |
| 权限预览 | 模拟用户查看有效权限范围 |

### 4.2 权限计算

```text
用户直接权限
  + 角色权限
  + 智能体绑定范围
  - deny 权限
  = 本次有效权限
```

运行时权限取“用户有效权限”和“智能体绑定范围”的交集。

### 4.3 输入输出

| 类型 | 内容 |
| --- | --- |
| 输入 | 用户、角色、权限配置、数据范围、智能体绑定范围 |
| 输出 | 有效权限、权限快照、权限拒绝记录 |

### 4.4 与其他模块的关联

| 关联模块 | 关联方式 |
| --- | --- |
| 用户与组织模块 | 权限以用户、角色、租户为基础 |
| 数据目录模块 | 字段权限依赖数据源、表、字段目录 |
| 工具配置模块 | 工具执行前必须校验工具和数据权限 |
| 智能体配置模块 | 智能体必须配置可使用用户或角色 |
| Workspace 模块 | 沙箱测试按模拟用户真实权限执行 |
| 智能体运行模块 | 构建 Prompt 前必须过滤无权限字段 |
| 审计模块 | 记录权限计算和权限拦截 |

### 4.5 扩展方向

后续可扩展行级权限、数据脱敏策略、审批授权、临时授权、权限有效期、权限变更影响分析。

| 附件上传 | 上传 Excel、图片、工艺文件、现场资料 |
| 需求提交 | 生成需求编号并进入需求池 |
| 需求补充 | 用户按开发端问题补充信息 |
| 需求确认 | 用户确认开发端整理后的需求摘要 |

### 5.2 输入输出

| 类型 | 内容 |
| --- | --- |
| 输入 | 模糊描述、模板表单、附件、补充说明 |
| 输出 | 原始需求单、结构化需求摘要、确认记录 |

### 5.3 与其他模块的关联

| 关联模块 | 关联方式 |
| --- | --- |
| 用户与组织模块 | 需求归属到用户、租户和部门 |
| 流程中心模块 | 需求提交后创建流程实例和任务 |
| 文件模块 | 需求附件保存到 MinIO/OSS |
| 开发端需求整理 | 需求内容用于生成 `需求.md` |

### 5.4 扩展方向

## 6. 流程中心模块

### 6.1 模块功能

流程中心负责把用户端、开发端、管理端串成完整业务闭环。

核心状态包括：

```text
草稿
已提交
需求整理中
待用户确认
已确认
评估中
已立项
数据准备中
配置开发中
沙箱测试中
待发布审核
已发布
用户试用中
验收中
已验收
已沉淀
```

核心功能包括：

| 功能 | 说明 |
| --- | --- |
| 状态流转 | 控制需求和项目当前阶段 |
| 任务分配 | 给开发者、FDE、管理员生成待办 |
| 审核控制 | 发布审核、验收审核、驳回重做 |
| 进度展示 | 给用户端展示当前状态和待处理事项 |
| 异常分支 | 支持退回修改、数据不足、测试不通过、审核驳回 |

### 6.2 输入输出

| 类型 | 内容 |
| --- | --- |
| 输入 | 需求提交、确认、评估、测试、审核、反馈动作 |
| 输出 | 流程实例、状态、待办任务、流程记录 |

### 6.3 与其他模块的关联

| 关联模块 | 关联方式 |
| --- | --- |
| 需求管理模块 | 需求提交后启动流程 |
| 开发端模块 | 流程任务驱动需求整理、数据评估、配置开发 |
| Workspace 模块 | 测试通过后才能进入发布审核 |
| 智能体配置模块 | 发布状态由流程审核控制 |
| 用户端模块 | 用户查看进度、确认需求、验收结果 |
| 审计模块 | 流程状态变化写入操作审计 |

### 6.4 扩展方向

后续可扩展可视化流程设计器、SLA 超时提醒、审批流配置、节点自动派单、项目看板、流程统计分析。

## 7. 数据源模块

### 7.1 模块功能

### 7.2 输入输出

| 类型 | 内容 |
| --- | --- |
| 输入 | 数据源类型、连接地址、账号、鉴权信息、同步频率 |
| 输出 | 可用数据源、连接测试结果、数据源状态 |

### 7.3 与其他模块的关联

| 关联模块 | 关联方式 |
| --- | --- |
| 数据目录模块 | 数据源配置后扫描表字段元数据 |
| 权限模块 | 数据源本身需要授权 |
| 工具配置模块 | SQL/HTTP 工具绑定具体数据源 |
| 智能体配置模块 | 智能体通过数据目录间接绑定数据源 |
| Hangfire 任务 | 数据扫描、数据同步、导入任务异步执行 |
| 审计模块 | 数据源新增、修改、测试、停用均记录审计 |

### 7.4 扩展方向

后续可扩展更多数据库类型、工业协议接入、实时流数据、数据质量监控、数据血缘、增量同步水位管理。

## 8. 数据目录模块

### 8.1 模块功能

数据目录模块把企业原始数据转化为 AI 可理解、权限可控制的结构化目录。

核心功能包括：

| 功能 | 说明 |
| --- | --- |
| 表字段扫描 | 扫描数据库或文件中的表、字段、类型 |
| 业务名称维护 | 给表和字段配置业务名称 |
| 字段注释维护 | 说明字段含义、单位、口径、取值范围 |
| 敏感级别配置 | public、internal、sensitive、confidential |
| AI 可见性配置 | 控制字段是否允许进入智能体上下文 |
| 目录发布检查 | 字段注释不足、权限缺失时提示风险 |

### 8.2 输入输出

| 类型 | 内容 |
| --- | --- |
| 输入 | 数据源元数据、人工补充注释、敏感级别、单位、示例值 |
| 输出 | 数据目录、字段说明、AI 可见范围 |

### 8.3 与其他模块的关联

| 关联模块 | 关联方式 |
| --- | --- |
| 数据源模块 | 从数据源扫描表字段 |
| 权限模块 | 字段权限依赖数据目录 |
| 规则模块 | 指标规则引用字段和口径 |
| 工具模块 | SQL 工具返回字段必须在目录内 |
| 智能体配置模块 | 智能体绑定可用数据目录 |
| 智能体运行模块 | ContextBuilder 只注入授权后的数据目录 |

### 8.4 扩展方向

后续可扩展字段语义识别、自动注释建议、数据质量评分、数据目录版本、指标目录、数据地图。

## 9. 规则配置模块

### 9.1 模块功能

规则配置模块用于沉淀业务口径，避免 AI 自行猜测指标和判断逻辑。

规则分为两类：

| 类型 | 说明 |
| --- | --- |
| 系统规则 | 平台统一规则，如权限拦截、敏感字段脱敏、输出格式 |
| 用户规则 | 企业或场景专属规则，如良率口径、成本口径、设备状态解释 |

规则类型包括：

| 类型 | 示例 |
| --- | --- |
| 指标规则 | 良率 = 合格数 / 总产出数 |
| 查询规则 | 质量问题先查日报，再查异常记录 |
| 映射规则 | 状态码 1 表示正常，2 表示异常 |
| 校验规则 | 日期范围不能超过 90 天 |
| 输出规则 | 财务数据保留两位小数 |
| 权限规则 | 敏感字段必须脱敏 |

### 9.2 输入输出

| 类型 | 内容 |
| --- | --- |
| 输入 | 规则名称、规则类型、规则表达式、引用字段、适用租户 |
| 输出 | 规则草稿、测试结果、发布规则版本 |

### 9.3 与其他模块的关联

| 关联模块 | 关联方式 |
| --- | --- |
| 数据目录模块 | 规则引用字段和业务口径 |
| 权限模块 | 规则执行范围受数据权限约束 |
| 工具模块 | Rule 工具调用已发布规则 |
| 智能体配置模块 | 智能体绑定可使用规则 |
| Workspace 模块 | 发布前测试规则是否正确 |
| 审计模块 | 规则修改、发布、调用都记录审计 |

### 9.4 扩展方向

后续可扩展规则依赖图、规则冲突检测、规则版本对比、规则推荐、低代码规则编排、复杂指标引擎。

## 11. 智能体配置模块


核心功能包括：

| 功能 | 说明 |

| 智能体基础信息 | 名称、编码、场景、描述 |
| 模型配置 | 主模型、备用模型、超时、重试 |
| 提示词配置 | 角色、任务边界、输出格式 |
| 数据绑定 | 绑定数据源、表、字段范围 |
| 规则绑定 | 绑定已发布规则 |
| 工具绑定 | 绑定已测试工具 |
| 用户授权 | 指定可使用用户或角色 |
| 版本管理 | 草稿、测试、发布、停用 |

### 11.2 输入输出

| 类型 | 内容 |
| --- | --- |
| 输入 | 模型、提示词、数据目录、规则、工具、授权用户 |
| 输出 | 智能体草稿版本、测试版本、发布版本 |

### 11.3 与其他模块的关联

## 12. Workspace 沙箱模块

### 12.1 模块功能

Workspace 沙箱模块负责在智能体发布前进行验证，确保能力可用、权限安全、输出符合业务口径。

核心功能包括：

| 功能 | 说明 |
| --- | --- |
| 模拟用户 | 选择不同用户身份测试权限 |
| Prompt 预览 | 查看权限过滤后的上下文 |
| 工具调用链路 | 查看调用了哪些工具、参数和结果 |
| 正常测试 | 验证常规问题能得到正确结果 |
| 异常测试 | 验证数据缺失、工具失败、模型失败场景 |
| 越权测试 | 验证无权限字段不进入 Prompt 和输出 |
| 边界测试 | 验证日期范围、空结果、大数据量、超时 |
| 测试报告 | 生成发布前测试记录 |

| 关联模块 | 关联方式 |
| --- | --- |
| 智能体配置模块 | 测试指定智能体版本 |
| 权限模块 | 按模拟用户计算真实权限 |
| 工具模块 | 调用工具并检查结果 |
| 规则模块 | 验证规则口径 |
| 流程中心模块 | 测试通过后进入发布审核 |

### 12.4 扩展方向

## 13. 智能体运行模块

### 13.1 模块功能

智能体运行模块是用户实际使用智能体时的执行核心。

运行流程：

```text
接收用户问题
  -> 校验用户和智能体权限
  -> 加载智能体发布版本
  -> 计算权限快照
  -> 构建安全上下文
  -> 调用模型
  -> 调用授权工具和规则
  -> AnswerGuard 输出检查
  -> 返回结果并记录审计
```

核心组件包括：

| 组件 | 说明 |
| --- | --- |
| AgentLoader | 加载智能体发布版本 |
| PermissionSnapshotService | 保存本次任务权限快照 |
| ContextBuilder | 构建授权后的 Prompt 上下文 |
| LLMAdapter | 调用主模型和备用模型 |
| ToolRuntime | 执行 SQL/HTTP/Rule 工具 |
| AnswerGuard | 检查越权字段、敏感信息和输出格式 |

### 13.2 输入输出

| 类型 | 内容 |
| --- | --- |
| 输入 | 用户 ID、智能体 ID、用户问题、入口来源 |
| 输出 | 智能体结果、权限说明、数据依据、trace_id |

### 13.3 与其他模块的关联

| 关联模块 | 关联方式 |
| --- | --- |
| 用户与组织模块 | 识别用户、租户和状态 |
| 权限模块 | 校验智能体权限和数据权限 |
| 智能体配置模块 | 加载发布版本 |
| 数据目录模块 | 注入授权数据目录 |
| 规则模块 | 注入授权规则 |
| 工具模块 | 执行授权工具 |
| 审计模块 | 记录完整执行链路 |
| 协同平台模块 | 飞书/钉钉消息通过任务进入运行模块 |

### 13.4 扩展方向

后续可扩展 Python AI Runtime、RAG 知识库、多轮任务记忆、复杂任务计划、模型网关、流式输出。

## 18. 核心功能总结

第一版核心功能可以总结为 8 件事：

| 序号 | 核心功能 | 说明 |
| --- | --- | --- |
| 1 | 承接用户需求 | 用户通过模板、表单、附件提交模糊业务需求 |
| 2 | 结构化需求 | 开发端把模糊需求整理成 `需求.md` 和确认摘要 |
| 3 | 配置企业数据 | 配置数据源、扫描数据目录、维护字段注释和敏感级别 |
| 4 | 控制访问权限 | 控制菜单、API、数据源、表、字段、工具、智能体权限 |
| 5 | 沉淀业务规则 | 将指标口径、查询口径、校验逻辑配置成规则 |
| 6 | 组装智能体能力 | 将模型、提示词、数据、规则、工具和授权范围组合成智能体 |
| 7 | 测试发布运行 | 通过 Workspace 沙箱测试和发布审核后供用户使用 |
| 8 | 审计反馈沉淀 | 每次运行可追踪，用户反馈后形成交付报告和复用模板 |

最终系统形成的主线是：

```text
需求可承接
数据可配置
权限可控制
规则可沉淀
工具可审计
智能体可发布
运行可追踪
效果可反馈
资产可复用
```

一句话总结：

```text
本系统第一版的核心，是建设一个把电子信息制造业业务需求转化为受控智能体能力的配置化平台。
```
